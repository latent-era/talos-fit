Fix two MEDIUM-severity build hygiene issues: the `allowBackup="true"` in the Android manifest (M6), and the commented-out ProGuard log-stripping rules (M7).

<context>
## M6: allowBackup="true" in manifest
- **File:** `androidApp/src/main/AndroidManifest.xml` (line 56)
- `android:allowBackup="true"` — future data stores backed up by default unless exclusion rules updated
- The app has its own Supabase sync — no need for cloud backup
- `dataExtractionRules` and `fullBackupContent` rules exist but don't cover future additions
- **Fix:** Set `android:allowBackup="false"`

## M7: ProGuard log-stripping rules commented out
- **File:** `androidApp/proguard-rules.pro` (lines 115-120)
- Log.d/v/i stripping rules are commented out
- Migration SQL and DB schema logged in release builds via `android.util.Log` in `DriverFactory.android.kt`
- **Fix:** Uncomment the rules to strip `Log.d`, `Log.v`, `Log.i` in release builds
- Keep `Log.w` and `Log.e` (warnings and errors should still be visible)
</context>
