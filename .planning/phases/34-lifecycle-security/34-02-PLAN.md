<objective>
Fix three HIGH-severity lifecycle issues: the unhandled workoutJob exception that silently loses workout data (H1), the unconditional DebugLogger in release builds (H9), and the ActivityHolder WeakReference lifecycle gap (H11).
</objective>

<context>
## H1: workoutJob has no top-level exception handler
- **File:** `shared/src/commonMain/kotlin/com/devil/phoenixproject/presentation/manager/ActiveSessionEngine.kt` (~line 1547)
- `coordinator.workoutJob = scope.launch { /* entire workout lifecycle */ }`
- No catch around the outer body — DB/IO exceptions propagate to SupervisorJob uncaught handler → crash on Android
- `saveWorkoutSession()` (~line 2035) has no try/catch
- **Fix:** Add top-level try/catch to workoutJob launch, emit error event, transition to Idle

## H9: DebugLogger unconditionally active in release builds
- **File:** `androidApp/src/main/kotlin/com/devil/phoenixproject/VitruvianApp.kt` (line 57)
- `newImageLoader()` passes `.logger(DebugLogger())` unconditionally
- Logs all image URLs to logcat in release builds
- **Fix:** Gate on `BuildConfig.DEBUG`: `.apply { if (BuildConfig.DEBUG) logger(DebugLogger()) }`

## H11: ActivityHolder static WeakReference lifecycle
- **File:** `shared/src/androidMain/kotlin/com/devil/phoenixproject/util/ScreenUtils.android.kt` (lines 11-18)
- `ActivityHolder` stores `WeakReference<Activity>` in singleton, set in `onCreate()`, never cleared in `onDestroy()`
- Between config change destroy and recreate, `getActivity()` returns null → silent no-ops
- **Fix:** Use `Application.registerActivityLifecycleCallbacks()` in `VitruvianApp` to maintain current activity reference
</context>

<tasks>
## Task 1: Add workoutJob exception handler (H1)
Wrap the workoutJob launch body in ActiveSessionEngine.kt with try/catch. Re-throw CancellationException. On other exceptions: log, emit error event via `coordinator._bleErrorEvents.tryEmit()`, transition to `WorkoutState.Idle`.

## Task 2: Gate DebugLogger on BuildConfig.DEBUG (H9)
In VitruvianApp.kt `newImageLoader()`, replace unconditional `.logger(DebugLogger())` with `.apply { if (BuildConfig.DEBUG) logger(DebugLogger()) }`.

## Task 3: Fix ActivityHolder lifecycle (H11)
In VitruvianApp.kt `onCreate()`, register `ActivityLifecycleCallbacks` that updates `ActivityHolder` on `onActivityResumed()` and clears on `onActivityPaused()`. Remove the manual `ActivityHolder.register()` from `MainActivity.onCreate()`.
</tasks>
