### Task 1: Add Supabase Entries to Info.plist

**File**: `iosApp/VitruvianPhoenix/VitruvianPhoenix/Info.plist`
**Action**: Add two new key-value entries before the closing `</dict>` tag:

```xml
<key>SUPABASE_URL</key>
<string>https://ilzlswmatadlnsuxatcv.supabase.co</string>
<key>SUPABASE_ANON_KEY</key>
<string>sb_publishable_UDrjasV6UJLm_IdIzGljoQ_YaRes4dQ</string>
```

**Note**: These are publishable keys (safe for client-side). The anon key starts with `sb_publishable_` which is Supabase's modern publishable key format.

**For CI/production builds**: These values can later be overridden via xcconfig files or build settings. For now, embedding in Info.plist matches the Android approach (local.properties → BuildConfig).

**Verify**: The plist is valid XML (well-formed, no duplicate keys).

**Done when**: Both `SUPABASE_URL` and `SUPABASE_ANON_KEY` entries exist in Info.plist.

### Task 2: Update PlatformModule.ios.kt to Read from NSBundle

**File**: `shared/src/iosMain/kotlin/com/devil/phoenixproject/di/PlatformModule.ios.kt`
**Action**: Replace the empty-string SupabaseConfig stub (lines 23-28) with NSBundle-based reading:

```kotlin
single {
    val bundle = platform.Foundation.NSBundle.mainBundle
    SupabaseConfig(
        url = bundle.objectForInfoDictionaryKey("SUPABASE_URL") as? String ?: "",
        anonKey = bundle.objectForInfoDictionaryKey("SUPABASE_ANON_KEY") as? String ?: ""
    )
}
```

**Key details**:
- `NSBundle.mainBundle` is already available in Kotlin/Native iOS targets
- `objectForInfoDictionaryKey` returns `Any?` — cast to `String?` with `as?`
- Fallback to empty string `?: ""` preserves graceful degradation if keys are missing
- `platform.Foundation.NSBundle` is the full Kotlin/Native import path
- The existing import for `platform.Foundation.NSUserDefaults` at line 20 confirms `platform.Foundation` is accessible

**Verify**: The file compiles without errors in the shared module's iosMain source set.

**Done when**: PlatformModule.ios.kt reads SupabaseConfig from Info.plist instead of empty strings.
