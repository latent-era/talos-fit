## Task 1: Fix perpetual re-push of NULL updatedAt sessions (M3)

**Action:** After a successful sync push, update `updatedAt` on all pushed sessions so they aren't re-sent.

**Specific changes:**
1. In `SyncManager.pushLocalChanges()` or after the push API call succeeds:
   ```kotlin
   // After successful push, stamp all pushed sessions with current time
   val now = Clock.System.now().toEpochMilliseconds()
   pushedSessions.forEach { session ->
       syncRepository.updateSessionTimestamp(session.id, now)
   }
   ```
2. If `SqlDelightSyncRepository` doesn't expose `updateSessionTimestamp`, add a method:
   ```kotlin
   fun updateSessionTimestamp(sessionId: String, timestamp: Long) {
       database.vitruvianDatabaseQueries.updateSessionTimestamp(timestamp, sessionId)
   }
   ```
3. Alternatively, add a batch update query to VitruvianDatabase.sq if iterating is too slow:
   ```sql
   updateSessionsTimestampBatch:
   UPDATE WorkoutSession SET updatedAt = :timestamp
   WHERE id IN :sessionIds AND updatedAt IS NULL;
   ```

**Verify:** After a sync push, grep/query confirms no sessions have NULL updatedAt. Build passes.

**Done when:** Locally-created sessions get updatedAt stamped after successful push, preventing re-push.

## Task 2: Handle Instant.parse() failure (M4)

**Action:** In `SyncManager.kt`, wrap the timestamp parsing in try/catch.

**Specific changes:**
1. Replace line ~106:
   ```kotlin
   val syncTimeEpoch = try {
       kotlin.time.Instant.parse(pushResponse.syncTime).toEpochMilliseconds()
   } catch (e: Exception) {
       Logger.w(e) { "Failed to parse syncTime '${pushResponse.syncTime}', using current time" }
       Clock.System.now().toEpochMilliseconds()
   }
   ```
2. This ensures sync always completes even with an unexpected timestamp format
3. The fallback to current time is safe — worst case, the next sync re-sends a few sessions that were already pushed (idempotent via upsert)

**Verify:** Build succeeds. Code review confirms the try/catch with sensible fallback.

**Done when:** Malformed syncTime from the Edge Function does not crash or hang the sync process.
